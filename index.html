Úvod

Proč je to důležité?

  • Designéři tráví až 30 % času předáváním práce a vysvětlováním kontextu
  • Claude Code propojuje Figmu přímo s kódem — bez mezikroků
  • Git vám dává kontrolu nad verzemi jako Figma branching, ale pro celý projekt
  • Agenti zvládnou rutinní úkoly paralelně — vy se soustředíte na design

Cíl: méně schůzek, méně back-and-forth, rychlejší handoff.

Představte si to jako Figma Auto Layout, ale pro celý vývojový proces — automatizace opakujících se kroků.

Základy

Co je Claude Code?

  • AI asistent v příkazové řádce (terminálu)
  • Rozumí kódu, Figma návrhům, i přirozenému jazyku
  • Umí číst, psát a upravovat soubory ve vašem projektu
  • Propojitelný s Figmou přes MCP (Model Context Protocol)

Co umí

Generovat kód z designu, opravovat chyby, spouštět testy, pracovat s Gitem

Co neumí

Nahradit vaše designové rozhodnutí — je to nástroj, ne náhrada

Je to jako mít junior vývojáře, kterému můžete říct co chcete v přirozeném jazyce.

Nastavení

Instalace Claude Code

  • Potřebujete: Node.js (v18+) a terminál
  • Mac: Terminal.app nebo iTerm2
  • Windows: PowerShell nebo Windows Terminal
# 1. Nainstalujte Claude Code
npm install -g @anthropic-ai/claude-code

# 2. Přejděte do svého projektu
cd ~/muj-projekt

# 3. Spusťte Claude Code
claude
Živá ukázka: otevřeme terminál, nainstalujeme a spustíme Claude Code.
Terminál = textová verze Finderu. Píšete příkazy místo klikání.

Figma integrace

Propojení Figmy s Claude Code

  • MCP = Most mezi Figmou a Claude Code
  • Claude vidí vaše návrhy, komponenty, tokeny
  • Generuje kód přímo z Figma komponent
# Instalace Figma MCP pluginu
/plugin install figma@claude-plugins-official

# Po instalaci se autentizujte přes prohlížeč
# (otevře se Figma přihlášení automaticky)

Co tím získáte

Pošlete Claude odkaz na Figma frame → dostanete funkční kód

Příklad použití

"Vezmi tento button z Figmy a vytvoř React komponentu"

Ukázka: nainstalujeme MCP, propojíme Figmu, vygenerujeme kód z komponenty.

Workflow

Figma-to-Code: Krok za krokem

1. Vyberte komponentu ve Figmě

Zkopírujte odkaz na frame nebo komponentu (pravý klik → Copy link)

2. Pošlete Claude příkaz

"Podívej se na tuto Figma komponentu a vytvoř z ní React kód"

3. Iterujte

"Uprav padding, přidej hover stav, použij naše design tokeny"

4. Hotovo — commitněte

Claude může rovnou uložit změny do Gitu

Jako když dáte feedback na design — ale místo komentáře ve Figmě píšete Claude přímo co chcete změnit.

Git pro designéry

Git: Verzování pro celý projekt

Git = systém verzí, jako historie ve Figmě, ale pro všechny soubory v projektu.

Figma konceptGit ekvivalentCo to dělá
Version Historygit logZobrazí historii změn
Branchgit branchVytvoří paralelní verzi
Save to Version Historygit commitUloží snapshot změn
Publish Branchgit pushNahraje na server
Update from Maingit pullStáhne nové změny
Pokud umíte Figma branching, rozumíte 80 % Gitu. Zbytek zvládne Claude za vás.

Git pro designéry

Git: Nastavení a základní příkazy

# Nastavení identity (jednorázově)
git config --global user.name "Vaše Jméno"
git config --global user.email "vas@email.cz"

# Základní workflow
git status              # Co se změnilo?
git add nazev-souboru   # Připravit ke commitu
git commit -m "Popis"   # Uložit snapshot
git push                # Nahrát na server

Spolupráce

GitHub: Kde se potkává design a kód

  • GitHub = sdílené úložiště kódu (jako Figma pro soubory)
  • Pull Request = návrh změny ke schválení (jako Figma review)
  • Issues = úkoly a bugy (jako Jira nebo Linear tickets)
  • Actions = automatické kontroly (jako Figma pluginy, ale pro kód)

Co potřebujete

  • GitHub účet (zdarma)

  • Přístup k repozitáři vašeho týmu

  • Claude Code umí vytvářet PR přímo z terminálu

Pull Request = "Podívejte se na mou změnu a schvalte ji" — stejný princip jako Figma review.

Agenti

Agent týmy: Paralelní práce

Claude může spustit více agentů najednou — každý řeší jiný úkol.

Pattern 1: Průzkum

Jeden agent analyzuje Figmu, druhý hledá existující komponenty v kódu

Pattern 2: Generování

Více agentů generuje kód pro různé komponenty současně

Pattern 3: Review

Agent kontroluje přístupnost, druhý ověřuje vizuální shodu s designem

Jako když máte tři asistenty — jeden exportuje assety, druhý píše specifikace, třetí kontroluje konzistenci.

Automatizace

Skills: Automatizace opakovaných úkolů

Skills = uložené příkazy pro věci, které děláte často. Jako Figma shortcuts, ale pro Claude.

1. Korekce textů

Automatická kontrola a oprava překlepů v UI textech

2. Přejmenování tokenů

Hromadný rename design tokenů v kódu i dokumentaci

3. Kontrola přístupnosti

Audit kontrastů, ARIA atributů a keyboard navigace

4. Formátování specifikací

Generování konzistentních handoff dokumentů z designu

5. Release notes

Automatické generování popisu změn pro každý release
